Portfolio Webpages, Frameworks, Web Tools.


- Stephen Beaty - 4/19/17

As I’ve said elsewhere on this site, it was important to me to create my website without using frameworks, libraries, or templates. However, a lot of those can be effective tools for web design and development. Bootstrap, for example, is great for quickly and easily giving a website lots of functionality and a standardized style that looks good. Other things, such as Sass, make coding with CSS easier. Sass is on my list of things to learn, and I may even incorporate it into my main site. With other tools, frameworks and libraries though, I’d like to make ‘example pages’. These would basically be mock ups for fictional websites, fictional companies, or whatever the case may be. I could use these to practice incorporating different tools. Or I could use them to try out some interesting JavaScript effects. It doesn’t make any sense to have lots of different animations on my main Sbeatycode site, but for a fictional educational children’s site? Why not. These would also be great to use as I learn more JavaScript frameworks, such as JSON, and as I continue to learn JQuery. Example sites that I make will be added to my portfolio. Maybe I’ll even add a new sub-page within my portfolio that just contains a lot of these pages.
