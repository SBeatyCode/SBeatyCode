Menubar Changes - click to close.


- Stephen Beaty – 4/29/17

I made some changes to the menubar, and some other minor changes to the site in general. I figured I would take some time and explain what I did.

One small issue I noticed was that in fullscreen (or at a window size close to fullscreen), on the Blog front-page, one of the four blog posts had a bug where it would ‘sink’ down below the other elements. At first I thought I had set the containers to be too wide, but after shrinking them a size it had the same problem. I didn’t want to shrink the container to be too small either, because that would destroy the look of the page. Thankfully the solution was simple. Since the posts are arranged in a 2x2 fashion, I merely had to add an extra container around each row. This was, the containers would be ‘stuck’ together, and one of the elements inside of the row containers couldn’t ‘sink’ away from the others.

Aside from fixing that blog issue, and fixing some small typos and word usage, most of the changes revolved around the top menubar. First I rearranged the items on the menu so that it makes more logical sense how it’s organized. I eliminated the blog menu bar entirely. Now, when the user hovers over the blog button in the menubar, the blog menu pops up underneath. I also got rid of the blog dropdown menu button for mobile, now it’s just one dropdown menu that includes the blog pages.

The thing that took me the longest to implement was being able to close the menu bar when the user clicked outside of the menu area. I keep trying different solutions, I kept experimenting, but I kept running into problems. Either the menu would never want to open because my menu-close function was making ALL clicks close the menu (even the ones meant to open it), or it would keep other parts of my script from functioning correctly, and so on. I spent time searching through documentation, looking up things online, but nothing seemed to work right. I had the idea to simply change it to a hover: element, but that brought along with it the ‘sticky’ hover element issues that mobile brings with it, so I decided against it.

It took me awhile to figure out how to make it so the user could click outside the menu to close it, while still keeping the ability to close it with the actual menu button. Eventually I came across an article over at css tricks by Philip Walton that addressed the issues I kept running into. This article is worth a read if you have the time. This is the code that was recommended:


$(document).on('click', function(event) {

if(!$(event.target).closest('#menuContainer').length) {

//code to hide the menu

}

});



After using the code that Walton suggested for closing a menu bar, the problem was fixed. While there’s definitely a lot of value in searching through documentation, experimenting, and trying all that you can on your own, there’s also nothing wrong with reaching out and trying to find help or advice. Chances are, if you’re having some kind of issue with coding, someone else out there in the world has also had that problem. The Internet is a great tool, provided that you don’t forget to experiment on your own as well – that’s important.
