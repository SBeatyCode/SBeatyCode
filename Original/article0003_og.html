CSS, and Responsiveness. The @media tag is my friend.


- Stephen Beaty – 4/19/17

While I was creating this site, I probably spent just as much time with, if not more, styling the CSS pages than the HTML. In Web Design, HTML serves as the ‘backbone’ of the webpage. It makes up the basic structure, and often contains the page’s content. CSS is integrally tied to HTML, and gives the webpage it’s design, style, flow, and with CSS3 even some functionality. Realistically speaking, you can’t have one without the other.

In building the site, I came upon several issues with the CSS. Finding a good overall layout, finding a ‘look’ for the site I liked, making the site responsive (or in other words, making it so the page looks good on different devices – including mobile), were all things that came up. Then of course, there’s making sure everything has a consistent, overall look across the website, while also making each page look ‘right’, so that the page’s appearance matched its function.

One very important factor for any website built now is responsiveness. This is achieved primarily through the use of CSS and JavaScript. While I was researching the best way to make my site responsive, there are 2 important things I came across. The first was using a CSS styling method that would inherently make a page responsive and the second is the @media tag.

I used the Grid-View method which you can read more about int he Grid View post. Without repeating myself too much, this is basically a way of styling a page by assigning elements a value of 1-12, with each row on a webpage having a total value of 12. What’s nice about this is that each numerical value is given a percent-value of how much space it takes up. So, when elements are placed inside of it and the page is resized, the elements within each of these numerical values are automatically scaled up and down with the page as it is resized.

One small thing I had to do to make sure this worked properly was to make sure the images were scaling with the page as well. Initially, once the page got to a small size, the images would extend out past where I wanted them to be. This is simply fixed though. By setting the max-width property (generally at 100%), setting the height to ‘auto’, and optionally setting a min-width, this helps the images to scale along with the rest of the elements.

Of course, all of this still gives the issue of making sure the site still looks good on mobile devices. Leaving it as it was, all the elements on the page just got too crowded and squished once the screen got too small. At first, I thought I was going to have to write out a script using JavaScript to resize and reorganize everything.

Then I found out about the @media CSS tag. This allows you to give the same elements different property values. By using ‘@media only screen and (max-width: = aValue) {property: value;}’, you’re able to change the properties of elements for whenever the screen the page is viewed on is less than the max value (represented above by aValue). When the site is viewed on a mobile device, or the browser window on a PC is resized to be small enough, these properties kick in.

By setting each Grid-View value to be the width of the whole page (or in other words setting each numerical value to have a width of 100%), each item takes up its own row. This gives the page a much cleaner look for smaller screens. By using some JavaScript, I was also able to reorganize and ‘hide’ different elements as well. Another thing I had to give attention to was changing the way the menu bars looked and behaved, so that they still looked ok. However, since I actually have one more change I still want to make to the menu bars, I’ll talk about these in more depth in an upcoming post.
